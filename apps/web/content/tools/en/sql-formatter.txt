[FAQ]
What does SQL formatter do?
A SQL formatter (SQL beautifier) takes messy or poorly formatted SQL queries and adds proper indentation, line breaks, and capitalization to make them readable.

Why format SQL?
Formatted SQL is easier to read, debug, and maintain. It helps identify missing keywords, misplaced clauses, and complex nested structures.

What SQL dialects are supported?
The formatter handles standard ANSI SQL and common variations. For specific database dialects (MySQL, PostgreSQL, Oracle), some syntax may vary.

How does it handle long WHERE clauses?
Long WHERE clauses are typically broken across multiple lines with proper AND/OR alignment for readability.

Can I customize keyword case?
Yes. Choose uppercase, lowercase, or title case for SQL keywords while preserving case for identifiers.

Does it validate SQL syntax?
The formatter focuses on formatting rather than validation. It may flag some obvious syntax issues but is not a full SQL validator.

Can I format subqueries?
Yes. The formatter handles nested subqueries and CTEs (Common Table Expressions) with proper indentation.

Is my SQL uploaded?
No. All formatting happens locally in your browser.
