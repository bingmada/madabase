[FAQ]
What is HTML encoding?
HTML encoding converts special characters into HTML entity codes. For example, < becomes &lt; and & becomes &amp;.

Why encode HTML?
Encoding prevents browsers from interpreting content as HTML. This is essential for displaying user input safely and preventing XSS attacks.

What characters are encoded?
Typically: <, >, &, ", and '. Some encoders also handle other special characters.

What is XSS?
XSS (Cross-Site Scripting) occurs when an attacker injects malicious scripts through unencoded user input. HTML encoding prevents this.

Can I encode for JavaScript instead of HTML?
No. This tool encodes for HTML. For JavaScript, use JavaScript string escaping which is different.

Is encoding the same as sanitizing?
No. Encoding converts characters to safe equivalents. Sanitizing removes or rejects dangerous content entirely.

When should I encode?
Encode whenever displaying user-generated content in HTML. This includes comments, messages, usernames, and any dynamic content.

Is my text uploaded?
No. All encoding happens locally in your browser.
