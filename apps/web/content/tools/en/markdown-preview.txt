[FAQ]
What is Markdown?
Markdown is a lightweight markup language that uses simple text syntax to format content. It was created by John Gruber in 2004 as a way to write structured text that is easy to read in plain form.

What Markdown syntax is supported?
This tool supports GitHub Flavored Markdown (GFM), including headers, emphasis, lists, links, images, code blocks, tables, task lists, and strikethrough.

How do I create headers?
Use # symbols: # for H1, ## for H2, ### for H3, and so on up to H6.

How do I create links and images?
Links: [text](url), Images: ![alt](url). Both can include titles: [text](url "title").

Can I create tables?
Yes. Use pipe symbols: | Header | Header | followed by | --- | --- | and then data rows.

How do I create code blocks?
Use triple backticks for code blocks with optional syntax highlighting. Use single backticks for inline code.

What is GitHub Flavored Markdown?
GFM is a variant of Markdown with extensions used by GitHub. It includes tables, task lists, autolinks, and strikethrough.

Is my Markdown stored anywhere?
No. All processing happens in your browser. Your Markdown content is never sent to any server or stored permanently.

Can I export the HTML?
Yes. You can copy the rendered HTML from the preview or export it for use elsewhere.
