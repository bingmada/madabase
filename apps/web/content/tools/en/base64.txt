[FAQ]
What is Base64 encoding?
Base64 is a binary-to-text encoding scheme that represents binary data in ASCII string format. It uses 64 different ASCII characters (A-Z, a-z, 0-9, +, /) to represent binary data, making it safe for text-based transmission and storage.

Why use Base64 encoding?
Base64 is used when binary data needs to be stored or transmitted as text. This is common in email (MIME encoding), web development (data URLs), APIs (binary data in JSON), and authentication (JWT tokens).

Can it handle Unicode and special characters?
Yes. The encoder uses UTF-8 encoding internally, so it correctly handles non-ASCII characters including Chinese, Japanese, emoji, and special symbols.

What is the difference between encode and decode?
Encoding converts plain text into Base64 format. Decoding reverses this process, converting Base64 back to the original plain text. Both operations are lossless and reversible.

Is my data uploaded to your servers?
No. All encoding and decoding happens locally in your browser using JavaScript. Your text never leaves your device, making this tool safe for sensitive data.

What characters does Base64 use?
Base64 uses 64 characters: uppercase A-Z, lowercase a-z, digits 0-9, plus (+), and forward slash (/). The equals (=) is used for padding.

Can I decode JWT tokens with this tool?
Yes. JWT tokens consist of three Base64-encoded parts separated by dots. You can decode the header and payload sections to inspect their contents.

Why does my decoded output look like gibberish?
This usually means you are trying to decode data that is not valid Base64, or you are decoding binary data that was not originally text. Some decoded content may appear as special characters or control codes.
