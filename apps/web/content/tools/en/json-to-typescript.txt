[FAQ]
What does JSON to TypeScript do?
It converts JSON data samples into TypeScript interface definitions. This provides type safety for working with JSON data in TypeScript projects.

What TypeScript types are generated?
The tool generates appropriate types: string, number, boolean for primitives, array types for arrays, and nested interfaces for objects.

Does it handle optional fields?
Yes. If a field is not present in all samples, it may be marked as optional with a ? suffix.

Can it handle arrays?
Yes. Arrays are typed with element types, like string[] or nested interface arrays.

What about null and undefined?
The tool analyzes the actual values in your JSON to determine appropriate types. Missing or null values are handled appropriately.

Is the generated code production-ready?
The generated interfaces are functional and can be used directly. You may want to add JSDoc comments or further customize for your needs.

Does it support discriminated unions?
This basic version generates standard interfaces. For complex union types, you would need to manually adjust the output.

Is my JSON uploaded?
No. All conversion happens locally in your browser.
