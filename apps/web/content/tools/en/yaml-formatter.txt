[FAQ]
What is YAML?
YAML (YAML Ain't Markup Language) is a human-readable data serialization format. It is commonly used for configuration files in DevOps tools like Docker, Kubernetes, and Ansible.

What does YAML formatter do?
A YAML formatter (YAML beautifier) takes messy or improperly indented YAML and reformats it with consistent indentation, proper spacing, and clean structure.

Why is YAML indentation important?
YAML uses indentation to define structure. Incorrect indentation changes the meaning of your YAML and can cause configuration errors. A formatter ensures consistent, correct indentation.

How does it handle nested structures?
The formatter preserves and properly indents nested objects and arrays, maintaining the correct YAML structure.

Does it validate YAML?
Yes. The formatter checks YAML syntax and reports any errors with line numbers to help you fix issues.

What common YAML errors does it catch?
Common errors include inconsistent indentation, incorrect list syntax, missing colons, and improper string quoting.

Can I format multi-document YAML?
Yes. Files with multiple YAML documents separated by --- are handled correctly.

Is my YAML uploaded?
No. All formatting and validation happens locally in your browser.
