[FAQ]
What does a URL parser do?
A URL parser breaks down a URL into its component parts: protocol, host, port, path, query string, and hash fragment.

What are URL components?
URLs consist of: scheme/protocol (https://), host (example.com), port (:8080), path (/path), query string (?key=value), and hash (#section).

What is the difference between path and query?
The path identifies the resource on the server (/products/123). The query string contains parameters (?id=123&color=blue).

What is a hash fragment?
The hash (#section) refers to a specific section within a page. It does not send a request to the server.

Why parse URLs?
Parsing helps with: building dynamic links, extracting parameters, validating URLs, and understanding URL structure.

Can I modify and reconstruct URLs?
This tool focuses on parsing. For building URLs, use URL construction methods in your programming language.

Is my URL data secure?
No. All parsing happens locally in your browser.
