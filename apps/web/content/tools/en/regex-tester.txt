[FAQ]
What is a regular expression?
A regular expression (regex) is a sequence of characters that defines a search pattern. It is used to match, find, and replace specific character combinations within text. Regex is supported in most programming languages and text editors.

What regex syntax does this tool support?
This tool supports JavaScript regex syntax. This includes character classes (\d, \w, \s), quantifiers (*, +, ?, {n,m}), anchors (^, $), groups ((), (?:)), and special characters.

How do I test for email validation?
A common email regex pattern is: /^[^\s@]+@[^\s@]+\.[^\s@]+$/. This matches most standard email formats but note that email validation is more complex than it appears.

Can I use capture groups?
Yes. You can use parentheses to create capture groups. For example, (\d+)-(\w+) would capture the digit and word parts separately. Use (?:...) for non-capturing groups.

How do I match special characters like dots or asterisks?
Use a backslash before the special character. For example, \. matches a literal dot, and \* matches a literal asterisk.

What does the g flag mean?
The "g" flag means "global" - it finds all matches rather than stopping at the first match. The "i" flag makes the search case-insensitive.

Is my regex and text uploaded anywhere?
No. All regex testing happens locally in your browser. Your patterns and text are never sent to any server.

Why is my regex not matching?
Check for escaping issues, character class errors, or incorrect quantifiers. Common problems include forgetting to escape special characters inside character classes, or using incorrect anchors.
