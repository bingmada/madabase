[INTRO]
URL 编码（也称为百分号编码）对于安全地在 URL 中传输数据至关重要。URL 只能包含有限的字符集（ASCII 字母、数字和一些特殊字符）。特殊字符、空格和非 ASCII 文本必须编码才能作为 URL 的一部分安全传输。

这款免费的在线 URL 编码解码工具处理可读文本和百分号编码格式之间的转换。无论你是在调试 API 请求、动态构建 URL 还是处理用户输入，这个工具都能让编码和解码变得轻松。

URL 编码用 "%" 后跟两个十六进制数字替换不安全的 ASCII 字符。例如，空格变成 %20，特殊字符如 & 变成 %26。这种编码由 RFC 3986 标准定义。

[FAQ]
什么是 URL 编码？
URL 编码（百分号编码）将字符转换为可以通过 Internet 传输的格式。它用 "%" 后跟两个十六进制数字替换不安全的字符。

为什么 URL 需要编码？
URL 只能包含有限的字符集。特殊字符（如空格、& 和 ?）必须编码以避免破坏 URL 结构。

哪些字符需要编码？
ASCII 范围之外的字符、空格和特殊字符（!、#、$、%、&、'、(、)、*、+ ,, /, :, ;, =, ?, @, [, ]）需要编码。

%20 是什么意思？
%20 是空格字符的 URL 编码表示。每个百分号后跟两个十六进制数字代表一个字节。

如何在 URL 中编码空格？
空格可以编码为 %20 或 +。两者在不同上下文（路径 vs 查询字符串）中都有效。

URL 编码和 HTML 编码一样吗？
不一样。URL 编码用于 URL 和查询字符串。HTML 编码用于在 HTML 文档中嵌入文本。两者都使用百分号编码但用途不同。

什么时候应该使用 URL 编码？
在构建包含用户输入的 URL、传递参数给 API 或调试 URL 相关问题时使用 URL 编码。

我的数据会上传吗？
不会。所有编码和解码都在浏览器本地进行。
